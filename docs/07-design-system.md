# 07-design-system.md

Version: 1.0

---

# Purpose

This document defines the visual design system of Aplet.

Every UI component must follow these rules.

Never create component-specific styles.

Always use the design system.

---

# Design Philosophy

Premium.

Minimal.

Business-first.

Professional.

Timeless.

Calm.

Structured.

Predictable.

---

# Visual Principles

Clarity before beauty.

Whitespace before decoration.

Hierarchy before color.

Content before effects.

Product before marketing.

---

# Color System

## Primary

Used only for:

- Primary CTA
- Active navigation
- Focus
- Links
- Important actions

Never use primary as decoration.

---

## Success

Used for

Revenue

Completed

Healthy

Verified

Growth

---

## Warning

Used for

Pending

Attention

Limited

Expiring

---

## Danger

Used for

Errors

Failures

Deleted

Blocked

Negative trends

---

## Neutral

Used for

Surfaces

Borders

Labels

Disabled

Secondary text

---

# Backgrounds

Application background

Very light gray.

Cards

White.

Never create colorful backgrounds.

---

# Typography

Only one font family.

Hierarchy

Display

H1

H2

H3

Body

Caption

Never invent new text sizes.

---

# Radius

Small

8px

Input

12px

Card

20px

Workspace

28px

Modal

32px

Never exceed 32px.

---

# Shadows

Three levels only.

Low

Cards

Medium

Dropdowns

High

Modal

Hero preview

Never stack multiple shadows.

---

# Borders

Very subtle.

Used only to separate surfaces.

Never rely on borders for hierarchy.

Spacing creates hierarchy.

---

# Icons

Single icon family.

Outline style.

Never mix filled and outline.

Never use emoji.

---

# Buttons

Variants

Primary

Secondary

Ghost

Danger

Link

Nothing else.

---

# Inputs

All inputs follow one structure.

Label

↓

Input

↓

Helper

↓

Validation

Never change order.

---

# Cards

Every card has

Title

↓

Primary Content

↓

Secondary Content

↓

Optional Action

Never place unrelated information together.

---

# Widgets

Widget anatomy

Header

↓

Metric

↓

Context

↓

Action

Widgets never explain.

Widgets summarize.

---

# Tables

Purpose

Operational data.

Never use tables for marketing.

Rules

Sticky header.

Row hover.

Sortable.

Searchable.

Paginated.

---

# Charts

Charts answer trends.

Not exact values.

Always provide context.

Never use more than five colors.

Avoid legends whenever possible.

---

# Empty States

Every empty state contains

Illustration (optional)

↓

Headline

↓

Explanation

↓

Primary Action

No jokes.

No decorative copy.

---

# Loading States

Prefer skeletons.

Never show spinners for page loads.

Only use spinners for inline actions.

---

# Error States

Explain

What happened.

Why.

How to recover.

Never expose technical errors.

---

# Notifications

Short.

Actionable.

Never exceed two lines.

Always include status.

---

# Modals

Use only when interruption is required.

Every modal must answer

What happened?

What happens next?

How can I cancel?

---

# Drawers

Use for editing.

Never use full-page modals.

---

# Responsive Rules

Desktop

Maximum information.

Tablet

Reduced density.

Mobile

Task-focused.

Never shrink desktop.

Rebuild layouts.

---

# Motion

Fast.

Natural.

Purposeful.

Motion communicates change.

Never decorate.

---

# Accessibility

AA contrast.

Keyboard navigation.

Visible focus.

Readable typography.

Large touch targets.

Never rely on color alone.

---

# Component Rules

Every component must

Solve one problem.

Have one responsibility.

Support loading.

Support empty.

Support error.

Support disabled.

Support responsive layouts.

Support RTL.

Support accessibility.

---

# Design Review Checklist

Before approving any component ask:

Does it solve one problem?

Is hierarchy obvious?

Can content breathe?

Can something be removed?

Would Stripe keep this?

Would Apple simplify this further?

If yes,

remove it.

---

# Final Principle

Users should notice their work.

Not the interface.

The interface disappears.

The business remains.
