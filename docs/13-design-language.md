# 13-design-language.md

Version: 1.0

---

# Purpose

This document defines the visual language of Aplet.

It is not a UI kit.

It defines how Aplet should feel.

Every design decision must reinforce this language.

---

# Brand Personality

Professional.

Reliable.

Intelligent.

Quiet.

Modern.

Premium.

Never playful.

Never flashy.

Never noisy.

---

# Emotional Goal

Users should feel

"I am managing a business."

Not

"I am using a website."

---

# Design Inspiration

Notion

Linear

Stripe Dashboard

Raycast

Vercel

Arc Browser

Apple Human Interface

GitHub

Never imitate.

Only learn principles.

---

# Design Keywords

Minimal

Calm

Confident

Structured

Technical

Readable

Efficient

Focused

Invisible

---

# What We Avoid

Marketing gradients.

Glassmorphism everywhere.

Neumorphism.

Heavy borders.

Rounded objects everywhere.

Huge icons.

Large illustrations.

Decorative patterns.

Shiny effects.

Fake dashboards.

Fake notifications.

Fake testimonials.

Meaningless charts.

---

# Information Density

Medium.

Not empty.

Not crowded.

Every screen should breathe.

Every element must justify its existence.

---

# Layout Philosophy

Whitespace is structure.

Alignment creates trust.

Grid creates rhythm.

Consistency creates confidence.

---

# Grid

Desktop

12 Columns

Tablet

8 Columns

Mobile

4 Columns

Never break the grid.

---

# Spacing Rhythm

8px Base Grid

8

16

24

32

48

64

96

128

Never invent spacing values.

---

# Surface Hierarchy

Level 0

Application Background

↓

Level 1

Sections

↓

Level 2

Cards

↓

Level 3

Floating Panels

↓

Level 4

Dialogs

Never exceed four visual layers.

---

# Visual Hierarchy

Large numbers.

Small labels.

Short explanations.

Actions last.

---

Example

Revenue

8.4M

+18%

Instead of

Revenue this month

You earned 8.4M Tomans

which is 18% higher than last month.

---

# Typography Rules

Never write paragraphs inside the product.

Maximum body length

Two lines.

Long explanations belong to documentation.

---

# Cards

Cards are containers.

Not decorations.

Each card answers one question.

If a card answers two questions,

split it.

---

# Dashboard Philosophy

Dashboard summarizes.

Analytics explains.

Do not mix them.

---

# Charts

Charts support numbers.

Charts never replace numbers.

Every chart needs context.

---

# Icons

Icons support labels.

Icons never replace labels.

Use icons sparingly.

---

# Motion

Motion communicates state.

Never decoration.

Fast.

Subtle.

Predictable.

---

# Empty Space

Empty space is intentional.

Never fill gaps because they look empty.

---

# Images

Prefer product.

Avoid stock photography.

Avoid people smiling at laptops.

If an image does not teach,

remove it.

---

# Landing Pages

Show product.

Not promises.

Show workflow.

Not marketing slogans.

Show interface.

Not illustrations.

---

# Workspace

Workspace is software.

Not a website.

Never use marketing components inside authenticated pages.

---

# Visual Noise

Remove

Extra badges

Extra labels

Extra borders

Extra icons

Extra colors

Extra shadows

Extra text

Extra buttons

Simplify first.

Then simplify again.

---

# Premium Quality Test

Before approving any screen ask:

Can one section be removed?

Can one sentence be removed?

Can one card be removed?

Can one color be removed?

Can one border be removed?

If yes,

remove it.

---

# Final Principle

The interface should disappear.

The user's business should become the focus.

If users remember the UI,

the design has failed.

If users remember how easy it was to complete their work,

the design has succeeded.
